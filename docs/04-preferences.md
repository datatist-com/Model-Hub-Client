# 04 - 作者偏好与约定（Frontend）

这份文档描述“我希望你继续维护时保持一致”的偏好与约定。它不是通用最佳实践，而是本项目当前的主观选择。

写给 AI 的一句话：把下面的 MUST 当成不可破坏的行为契约；不确定就不要改它。

## 1) 体验偏好（优先级最高）

- Header 会话标签（Session Tabs）必须像“浏览器标签”一样可依赖：
  - 切换不丢状态（keep-alive）
  - 可关闭，并且关闭要驱逐缓存释放内存
  - 页面内 drill-down/返回场景优先使用 replace-tab（替换当前标签，不要新增标签）
- 会话标题格式保持克制：`页面名 (当前层名称)`，不要叠很多层。

验收（每次改动都要能自证）：

- 能打开多个会话、切换不丢状态
- 能关闭会话且不会“刷新复活”
- drill-down/返回默认不会新增会话

## 2) 主题偏好

- Dark-first：深色是默认体验
- Light 模式强调可读性：
  - 避免淡青/荧光类 hover/选中效果，尽量用中性灰
  - 表格/标题/分页在 light 模式下必须有足够对比度
- 主题切换要“顺滑”：表格表头/背景色切换应有过渡（避免闪变）

禁止项（不要做）：

- light 模式使用淡青/荧光色作为主要 hover/选中反馈
- 为了“好看”牺牲可读性（尤其是表格/分页/标题）

样式主要集中在：

- `src/styles.css`

## 3) i18n 偏好

- UI 文案一律用 i18n key，不要写死中文/英文
- 与会话标签相关的标题/菜单项：存储时优先存 `labelKey`，展示时 `t(labelKey)`，避免刷新后显示原始文本

禁止项（不要做）：

- 把 UI 文案写死在组件里（除非是临时 debug 且最终会删）
- 为了省事只补中文不补英文（除非用户明确只要中文）

资源文件：

- `src/locales/zh-CN.ts`
- `src/locales/en-US.ts`

## 4) 导航偏好（避免“标签爆炸”）

- 列表 → 详情 → 返回：默认 replace-tab（同一条工作流在一个标签内完成）
- 只有当用户明确需要并行处理两条对象（例如同时编辑两个用户）时，才允许新开第二个会话标签

AI 执行提示：

- “并行处理”必须有明确用户需求；否则默认 replace-tab
- 并行会话的区分字段必须进入 URL（通常是 query）

## 5) 可访问性/键盘细节

- 会话标签的关闭按钮不应进入 Tab 顺序（避免键盘焦点很丑）
- 交互控件的 hover/active 反馈要清晰但不过度抢眼

## 6) 代码风格偏好（简版）

- TypeScript：优先显式类型于关键边界（storage、router state、API DTO）
- 组件拆分：优先“少而清晰”，不要为了复用过度抽象
- 改动要可回滚：避免一次 PR 同时重构 + 加功能

交付偏好：

- 改动集中在最相关的文件，避免“顺手清理”
- 最终交付说明里写清楚：改了什么、如何验证、回归点是什么

如果你需要新增一条偏好，请在本文件追加，并在 PR 描述中说明为什么。
